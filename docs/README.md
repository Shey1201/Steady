# Steady 文档总览

这里存放 Steady 的项目文档。根目录 README 负责快速介绍项目，本目录负责记录更完整的设计、开发、部署和维护信息。

## 文档列表

- [产品说明](./PRODUCT.md)：项目定位、用户场景、核心体验闭环。
- [架构说明](./ARCHITECTURE.md)：多端目录结构、模块边界、数据流和后续拆包方向。
- [开发指南](./DEVELOPMENT.md)：本地开发、脚本说明、测试和常见问题。
- [环境变量](./ENVIRONMENT.md)：本地与部署环境需要配置的变量。
- [API 文档](./API.md)：当前 Serverless API 的路径、请求和响应说明。
- [部署指南](./DEPLOYMENT.md)：Vercel 和桌面端构建注意事项。
- [贡献指南](./CONTRIBUTING.md)：分支、提交、测试和文档维护建议。
- [变更记录](./CHANGELOG.md)：记录重要结构调整和版本变化。
- [路线图](./ROADMAP.md)：后续迭代优先级与待完善能力。

## 维护原则

- README 保持简洁，面向第一次访问项目的人。
- `docs/` 写清楚开发和维护细节，面向项目参与者。
- 文档应尽量跟随真实代码状态更新，避免写成尚未实现的能力。
